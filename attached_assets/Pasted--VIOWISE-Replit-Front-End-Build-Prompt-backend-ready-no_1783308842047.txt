## BUILD PHILOSOPHY — read first
Build a complete, premium React front end for VIOWISE. All data is MOCKED for now, BUT architect it so a real backend (Supabase/REST) can replace the mock with minimal changes:
- Put ALL data access behind a single service layer (e.g. `src/services/api.ts`) with async functions: getUsers(), getPosts(), createPost(), getMatches(userId), requestCall(from,to), getRequests(userId), respondRequest(id,action), getStorySummary(context), getMatchReason(a,b), logInteraction(event). Today these return mock data from `src/data/seed.ts` (with small await/delay to mimic network). Later, only this file changes to call the backend — components never change.
- Keep all types in `src/types.ts` (User, Post, Request, Interaction) so backend and frontend share one contract.
- Use React Context for session state (current user, role) — shaped so it can later be backed by real auth.
- Do NOT hardcode data inside components. Everything reads through the service layer.
- Build efficiently: ONE shared component each for Nav, Card, Button, Toggle, Avatar, Modal, AccessibilityControl. One global stylesheet with CSS variables. Don't duplicate styles per page. Output short summaries after each phase, not full files. Edit surgically; don't regenerate working files. Ask no questions — all specs below.
